# Benchmark

The benchmark contains 459 case directories. A case is the unit of evaluation used by the model comparison, retrieval configuration, agent ablation, and optimization ablation experiments.

Each case directory contains:

- `CaseNNN_Base.aadl`: the AADL model submitted to the pipeline;
- `CaseNNN_Base.txt`: the same model in text form for loaders that read plain text;
- `CaseNNN_Req.txt`: the natural-language requirement;
- `CaseNNN_Req_Expected.json`: the expected requirement-analysis reference used for inspection and debugging.

`cases_manifest.csv` lists all released cases and file sizes. Case reports generated by experiments are stored under a result root, not under the benchmark input directory.
