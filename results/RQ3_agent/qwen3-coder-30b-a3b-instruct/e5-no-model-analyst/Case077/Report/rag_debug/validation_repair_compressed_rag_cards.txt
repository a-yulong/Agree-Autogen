[KSYN-1]
Source: processed\ksyn\aadl_scope_notes.md
Topic: aadl_scope_notes
AGREE assumptions and guarantees that refer to component ports should be placed in valid component type scopes. Implementation-level bindings should be placed in implementation scopes after relevant subcomponents and connections. Subcomponent ports should not be used in a component type contract unless represented through valid variables or outputs.

[KSYN-2]
Source: processed\ksyn\agree_syntax_notes.md
Topic: agree_syntax_notes
AGREE contracts are written inside `annex agree {** ... **};` blocks. Component type annexes are the normal location for assumptions, guarantees, constants, and equation declarations. Component implementation annexes are used for implementation-level bindings such as assignments when they are valid in that scope.

[KSYN-3]
Source: processed\ksyn\aadl_scope_notes.md
Topic: aadl_scope_notes
AADL package references require matching `with` clauses. Generated AGREE code should not introduce package-qualified identifiers, data types, or property references that are absent from the analyzed model and its configured dependencies.

</KSYN_CARDS>

Topic: implementation scope output assign
For implementation contracts, use assign only when the left-hand side is an existing output feature or declared auxiliary variable. Do not place guarantee statements in implementation scope.

[KDEF-1]
Source: processed\kdef\defensive_rules.jsonl
Topic: Named Thing Has No Type
If AGREE reports "named thing must be an expression with a type" or a guarantee expression has type <error>, trace the first unresolved identifier or invalid helper declaration on the same line. Fix the root unresolved reference, or delete the invalid clause instead of adding architecture. Recommended pattern: Use only visible AADL features or eq helpers declared in the same annex; otherwise remove the guarantee.

[KDEF-2]
Source: processed\kdef\defensive_rules.jsonl
Topic: AGREE Scalar Type Names
If AGREE reports "Couldn't resolve reference to 'integer'" or "Couldn't resolve reference to 'boolean'" in a const or eq declaration, the generated AGREE type name is invalid. AGREE scalar declaration types are int, real, and bool. Recommended pattern: const lower_limit : int = 0; eq ok : bool = x >= lower_limit;

[KDEF-3]
Source: processed\kdef\defensive_rules.jsonl
Topic: Implementation Scope Guarantee Error
If diagnostics point to implementation-level references or subcomponent paths in a component type annex, move only valid implementation bindings to the implementation annex or delete the invalid guarantee. Component type contracts may reference only features visible at that type scope. Recommended pattern: Use a type-level guarantee over visible ports, or an implementation-level assign when the left-hand side is a valid output.
